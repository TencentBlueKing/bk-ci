 # 项目属性、权限与分片

## 项目属性

项目属性通常承载：

- 流水线方言或 PAC 相关设置
- 项目限制项
- 可授权范围
- 某些组织或业务线信息

所以项目属性不是简单附加字段，而是会影响多个下游模块行为的配置面。

## 权限与成员

项目层权限常见问题包括：

- 成员能不能访问项目
- 某类资源是否允许在项目下创建
- 项目启停后权限边界是否变化

排查时先分清是：

- 项目不存在
- 项目状态不允许
- 成员没有项目级权限
- 资源模块又叠加了一层自身权限

## 项目分片与路由

项目模块还可能承担分片路由职责，例如：

- 某些项目映射到不同数据源或分区
- 项目级规则影响请求最终路由

这类问题的常见症状：

- 同样请求在不同项目结果不同
- 项目迁移后读写落点不一致
- 数据看起来“丢了”，实际上是路由错了

## 推荐排查顺序

1. 先确认项目英文标识是否正确
2. 再确认项目状态、属性和成员权限
3. 再确认是否存在项目分片或特殊路由
4. 最后再去下游模块看资源本身逻辑
