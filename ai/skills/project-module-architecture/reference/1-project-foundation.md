# 项目基础心智

## Project 为什么重要

`Project` 是 BK-CI 的基础模块，因为几乎所有核心资源都以项目为归属边界：

- 流水线
- 代码库
- 凭证
- 构建环境
- 制品

很多模块都会接收 `projectId`，但项目语义应以 `Project` 模块为准。

## projectId 的真实含义

这是最容易踩坑的点之一：

- 数据库里可能有一个内部 UUID 字段
- 但业务接口里大量出现的 `projectId`，通常实际对应项目英文标识

所以当你看到别的模块传 `projectId` 时，先别默认它是数据库主键。

## Project 模块常见职责

- 项目创建、更新、删除、启停
- 项目属性与扩展配置
- 项目标签与组织属性
- 项目审批状态
- 项目级权限与成员边界
- 项目分片和路由基础能力

## 常见判断法

如果问题是下面这些，更可能属于 `Project`：

- 项目找不到 / 状态不对
- 项目属性不生效
- 某资源在项目维度下不见了
- 同一个 `projectId` 在不同服务语义不一致

如果问题已经明确落在流水线、代码库或制品资源本身，就切到对应模块 skill。
