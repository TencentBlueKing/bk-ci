# BkRepo、文件任务与扩展

## 存储后端

常见有两类后端：

- BkRepo
- 本地磁盘

排查前先确认实际命中的是哪套实现，否则看错代码会非常快地走偏。

## 文件任务

文件任务常见出现在：

- 上传后处理
- 异步归档
- 构建完成后的制品整理
- 某些清理或迁移动作

如果接口已经成功，但最终结果不一致，优先看任务状态和消息监听。

## BkRepo 集成

BkRepo 问题常见包括：

- 上传成功但查询不到
- 元数据存在但真实文件缺失
- 下载鉴权失败
- 商店归档与普通归档行为不同

这类问题通常要同时确认：

1. 后端实现类
2. 元数据是否落库
3. 任务是否完成
4. 下载或查询入口是否正确

## 扩展与排查

如果要扩展制品能力，至少要检查：

- 文件类型枚举
- 路径规则
- 元数据字段
- 存储实现
- 下载与清理链路

推荐排查顺序：

1. 看上传或归档入口
2. 看元数据落库
3. 看任务状态
4. 看后端真实文件
5. 最后看下载和查询行为
