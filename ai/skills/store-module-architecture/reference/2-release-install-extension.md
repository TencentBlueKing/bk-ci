# 发布、安装与扩展

## 发布链路

商店组件发布通常会经历：

- 提交
- 构建
- 测试
- 审核
- 发布 / 上架
- 下架或停用

不同组件类型的具体步骤可能不同，但“状态流转”是排查核心。

## 安装与升级

安装问题常见包括：

- 安装成功但资源未完全落地
- 升级成功但旧实例未同步
- 目标项目权限或依赖条件不满足

因此安装问题不要只看商店接口返回，还要看目标模块是否真正接住了组件。

## 扩展点

Store 往往包含大量扩展点，例如：

- 处理器链
- 工厂类
- 审核或后置处理
- 统计和搜索增强

当你扩展商店能力时，优先判断你改的是：

- 组件主生命周期
- 市场展示
- 安装链路
- 还是审核和统计

## 推荐排查顺序

1. 先看组件当前状态
2. 再看发布或安装链路走到哪一步
3. 再看是否是审核、权限或目标项目接入问题
4. 最后看扩展点和后置处理是否把结果改坏了
