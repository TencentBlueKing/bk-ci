# 商店基础与组件类型

## Store 管什么

`Store` 模块关注的是组件市场化能力，包括：

- 组件发布
- 组件审核
- 安装和升级
- 分类、标签、成员与评论
- 统计与市场展示

## 常见组件类型

常见有三类：

- 插件（Atom）
- 模板（Template）
- 镜像（Image）

这三类虽然共享市场逻辑，但各自仍有不同的数据结构、状态流转和安装目标。

## 为什么容易重复

Store 很容易和下面模块重叠：

- 模板模块
- 插件开发模块
- 制品归档模块

判断原则是：

- 问题是“组件如何进入市场和被消费” -> 看 `Store`
- 问题是“组件本身如何定义和运行” -> 看对应模块

## 常见数据与附属能力

商店里除了主组件表，通常还会有：

- 分类表
- 标签表
- 成员表
- 评论与点赞
- 统计与审批

因此商店问题常常不只是“发布失败”，而是某个辅助能力影响了整体结果。
