# 数据访问、事务与异常处理

## 数据访问重点

- 先确认存储边界，再决定 Repository / DAO 设计
- 查询优化、索引和批量操作要从真实访问路径出发
- 数据访问安全和敏感数据处理不能靠默认侥幸

## 事务重点

- 事务传播行为要匹配业务边界
- 查询操作优先只读事务
- 超时、重试和幂等要一起考虑

## 异常处理重点

- 业务异常、系统异常、外部依赖异常要分层
- 全局异常处理保证输出一致
- 日志要记录上下文，但避免泄露敏感信息

## 常见坑

- 事务边界模糊，导致副作用不可控
- 把底层异常直接暴露给上层调用者
- 只有异常抛出，没有统一响应和观测信息
