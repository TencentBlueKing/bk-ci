# 配置管理与 Bean 装配

## 配置管理重点

- 主配置与环境配置分离
- 敏感信息不硬编码，优先走环境变量或配置中心
- 配置项命名保持一致，类型安全绑定优先

## Bean 装配重点

- 默认使用构造器注入
- 复杂 Bean 用 `@Configuration` + `@Bean`
- 条件装配使用 `@ConditionalOn*`
- 多实现时用 `@Qualifier` 或 `@Primary`

## 生命周期与边界

- 初始化和销毁逻辑要明确
- 异步初始化和资源清理要可控
- 避免循环依赖，必要时重新拆分职责

## 常见坑

- 配置散落、优先级不清
- 为了快使用字段注入，后续测试和维护变差
- 条件装配过多但没有统一设计
