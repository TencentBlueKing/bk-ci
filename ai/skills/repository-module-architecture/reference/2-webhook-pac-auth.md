# Webhook、PAC 与认证

## 认证

仓库认证常见方式包括：

- HTTP / HTTPS
- SSH
- OAuth
- Token
- GitHub App

问题排查时先确认：

1. 仓库类型
2. 认证方式
3. 凭证是否匹配该仓库类型

## Webhook

Webhook 常见问题包括：

- Webhook 没触发
- 触发了但事件内容不对
- 同一个仓库在不同平台表现不同

排查时先分清：

- 仓库侧 Webhook 是否配置成功
- 平台侧是否正确接收和路由
- 事件是不是被 PAC / 触发器后续过滤掉了

## PAC

仓库侧 PAC 主要负责：

- 允许代码库中的流水线定义参与平台流程
- 为后续 YAML 读取、分支、路径和触发提供仓库上下文

所以 PAC 问题不要只看仓库模块，也要同步检查：

- YAML 转换
- 模板体系
- 触发器配置

## 推荐排查顺序

1. 仓库保存是否正确
2. 认证是否有效
3. Webhook 是否成功
4. PAC 开关与上下文是否正确
5. 最后再看 YAML / 模板 / 执行链路
