# 运行形态与升级

## Agent 在机器上的角色

- Daemon 负责守护
- Agent 主进程负责与服务端持续交互
- Worker 由 Agent 按需拉起执行具体任务

## 先理解的三件事

1. 构建机上有哪些常驻进程
2. 进程之间如何拉起和守护
3. 升级发生时如何与正在执行的任务共存

## 升级心智

- Agent 升级不只是替换二进制，还影响 Worker、JDK 或第三方组件
- 升级逻辑要考虑是否有运行中任务
- 稳定升级比激进升级更重要

## 典型排查点

- Daemon 是否还在正常守护 Agent
- Agent 是否因为锁、权限或运行目录问题拉起失败
- 升级后是否出现版本不兼容或组件缺失
