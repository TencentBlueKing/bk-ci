# 文件结构、格式与导入

## 重点关注

- 文件名与顶级声明保持一致
- 源文件使用 UTF-8
- 缩进使用 4 个空格，不使用 Tab
- 单行长度控制在 120 以内
- 保持 import 简洁，不使用 wildcard import

## 文件结构心智

- 单文件单主职责优先
- 顶级声明过多时，文件名要能准确表达内容
- 类、函数、属性之间用空行分组，帮助快速扫读

## 格式规则里最值得保留的约束

- 长调用链在 `.` 前换行
- 长参数列表一参一行
- getter / setter 分行
- 有换行的控制块使用大括号

## 常见坑

- 行宽超标后用不稳定方式硬折行
- import 堆积、保留未使用导入
- 一个文件同时承载太多无关顶级声明
