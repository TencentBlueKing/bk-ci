# 命名、函数设计与对象调用

## 命名重点

- 包名全部小写，不用下划线
- 类名使用 PascalCase
- 函数、属性、局部变量使用 lowerCamelCase
- 布尔值优先用 `is`、`has`、`can`、`should`

## 函数设计重点

- 保持单一职责，避免深层嵌套
- 优先早返回、表达式函数、提取小函数
- 默认参数和命名参数用于提升可读性，而不是制造形式主义

## BK-CI 项目里的高价值约定

- 多参数且容易混淆的调用，优先使用命名参数
- data class 实例化优先显式命名参数
- 不要直接写完整类路径再调用方法，先 import 再用简洁类名

## 常见坑

- 为了省事用位置参数传多个同类值，后续阅读和维护困难
- 把类做成巨型工具箱，函数和数据职责混杂
- 过度强调形式，反而忽略调用意图是否清晰
