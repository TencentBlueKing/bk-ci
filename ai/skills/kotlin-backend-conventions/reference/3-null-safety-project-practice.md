# 空安全、集合与 BK-CI 项目约定

## 空安全

- 优先用 `?.`、`?:` 和类型系统表达可空性
- `!!` 只能在确有充分理由时使用
- 失败场景优先用明确异常或结果类型表达

## 集合与函数式写法

- 优先不可变集合
- 合理使用 `map`、`filter`、`fold`、作用域函数
- 简单 lambda 用 `it`，复杂场景显式命名参数

## 项目协作约定

- Kotlin 规范要和 BK-CI 后端分层、测试、数据库和 API 规范一起看
- 语言层规范不替代业务边界判断
- 质量门禁和静态检查能托底的内容，优先交给工具

## 常见坑

- 把所有风格问题都堆给 always rule
- 把 Kotlin 语言规范和 Spring Boot / 微服务架构手册混成一个文档
- 只谈语法，不谈项目里真正高频出错的约束
