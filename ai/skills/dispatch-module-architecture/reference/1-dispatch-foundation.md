# 模块结构与核心对象

## Dispatch 的角色

- 接住来自流程引擎的调度请求
- 根据资源类型、策略和状态选择执行环境
- 推进任务从排队到被领取再到完成

## 理解模块时先分三层

1. API / listener：请求入口和事件推进
2. service：调度决策和业务规则
3. dao / model：状态持久化与资源记录

## 核心对象心智

- 调度记录：描述任务当前处于哪个资源类型和状态
- 资源对象：第三方构建机、Docker、Kubernetes 等执行环境
- 队列对象：描述等待中的任务集合
- 配额对象：限制项目级、系统级并发和时长

## 适合优先阅读的主题

- 第三方构建机调度主链路
- 队列推进服务
- 配额检查逻辑
- 资源类型到调度实现的映射
