# 队列、配额与调度策略

## 排查调度问题的第一步

- 先判断任务是没进队列、进队列后没推进，还是推进后没有可用资源

## 队列心智

- 队列不是单纯缓存，而是调度推进的中间状态
- 队列服务通常和监听器、构建机状态、环境匹配一起工作
- 同一个任务在不同阶段可能对应不同状态记录

## 配额心智

- 项目配额控制单项目资源占用
- 系统配额控制全局资源上限
- 配额命中时，任务可能保持排队而不是失败

## 选择构建机时通常会考虑

- 资源类型是否匹配
- 环境 / 标签 / 配置是否匹配
- 当前是否有空闲资源
- 配额是否允许继续下发

## 常见坑

- 只看资源池有机器，却没看配额是否已满
- 只看队列表，不看监听器是否正常推进
- 调度策略变了，但状态流转和回收逻辑没同步更新
