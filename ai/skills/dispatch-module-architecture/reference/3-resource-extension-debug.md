# 资源类型扩展与排查

## 什么时候算新增调度类型

- 引入新的执行资源形态
- 新增新的构建机选择策略
- 针对已有资源类型增加独立调度分支

## 扩展时至少同步看的点

- API / 事件入口是否能接到新类型
- service 是否有对应调度决策
- model / dao 是否有状态与资源记录
- 队列、配额、回收和清理逻辑是否同步支持

## 调试建议

- 先用单一资源类型验证主链路
- 再验证资源不足、配额命中、取消和失败回收
- 最后再看与 Agent / Worker 的边界是否清晰

## 边界提醒

- Dispatch 决定任务去哪跑
- Agent 决定构建机如何接任务和拉起执行器
- Worker 决定任务如何真正跑起来
