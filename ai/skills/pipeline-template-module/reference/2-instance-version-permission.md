# 实例化、版本与权限

## 实例化

模板实例化关心的是：

- 模板如何生成真实流水线
- 模板参数如何传入
- 模板实例与模板版本之间如何关联
- 批量实例化时如何控制一致性与失败处理

常见问题表现：

- 实例化成功但结果结构不对
- 部分实例化成功，部分失败
- 模板更新后实例没有正确同步

## 批量操作

批量实例化或批量更新会放大模板设计问题，排查时优先看：

1. 模板版本是否稳定
2. 参数映射是否兼容所有实例
3. 是否存在项目权限差异
4. 某些实例是否已经偏离模板定义

## 版本管理

模板版本常见语义包括：

- 草稿
- 正式发布版本
- 分支版本
- 历史版本

高风险点：

- 只改最新版本逻辑，不考虑历史版本回读
- 草稿和正式版本状态切换不清晰
- 分支版本和正式版本混淆

## 权限控制

模板权限通常不仅是“能不能看”，还包括：

- 能否创建模板
- 能否编辑模板
- 能否实例化模板
- 能否跨项目使用模板
- 能否发布到商店或从商店安装

所以当用户说“模板不能用”时，先确认是：

- 模板本身不存在
- 版本不对
- 参数不匹配
- 还是权限不够

## 推荐排查顺序

1. 先确认模板 ID、模板版本、实例类型
2. 再确认实例化入口落在 V1 还是 V2
3. 再确认权限校验是否通过
4. 最后看实例化后的流水线资源是否正确落库
