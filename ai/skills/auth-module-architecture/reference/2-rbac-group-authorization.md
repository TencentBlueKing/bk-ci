# RBAC、用户组与授权链路

## RBAC 的基本心智

- 用户通常不直接拿权限，而是通过用户组获得权限
- 资源权限通常是“组 + 动作 + 资源”组合出来的
- 默认组、分级管理员、资源组配置会直接影响授权体验

## 排查权限问题时先看

1. 用户是否在正确的组里
2. 组是否拥有目标动作
3. 目标资源是否已被正确注册或同步
4. 校验时传入的资源类型、资源 ID、动作是否正确

## 常见授权链路问题

- 默认组配置和实际动作定义不一致
- 资源已存在，但组权限没同步
- 迁移后用户组和历史授权数据错位
