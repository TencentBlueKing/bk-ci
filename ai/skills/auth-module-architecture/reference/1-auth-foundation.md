# 模块结构与核心对象

## Auth 的核心职责

- 统一管理资源、动作、用户组、成员和授权关系
- 提供权限校验、授权申请、资源同步和认证能力
- 作为 BK-CI 权限平台与 IAM 的桥梁

## 理解模块时先分三层

1. API / 回调 / 接口入口
2. RBAC Provider 与业务服务
3. DAO / 模型 / 权限数据存储

## 核心对象心智

- 资源：被保护的业务对象
- 动作：对资源能做什么
- 用户组：权限授予的主要载体
- 成员：用户或主体与组的关系
- 授权：组、动作、资源的组合结果

## 适合先理解的点

- 资源和动作如何落表
- 用户组如何组织默认权限
- 校验请求如何一路走到 RBAC 服务
