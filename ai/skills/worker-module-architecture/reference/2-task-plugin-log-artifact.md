# 插件执行、日志与制品

## 先分清三类能力

- 任务执行：根据任务类型创建并运行执行器
- 日志服务：收集、缓冲、过滤并上报日志
- 制品处理：把构建产物归档到制品库

## 插件执行心智

- Worker 会根据任务类型创建执行器
- Market Atom 这类插件任务通常在 Worker 里真正落地执行
- 如果插件定义、字段或输入输出异常，仍要回看 `pipeline-plugin-development`

## 日志排查重点

- 日志是否成功进入 Worker
- 日志是否在本地处理时被过滤或折叠
- 日志是否成功进入上传队列
- 服务端看到的日志与 Worker 本地日志是否一致

## 制品排查重点

- 产物是否真的生成在预期工作区
- 归档任务是否拿到了正确路径和上下文
- 失败发生在本地文件、上传请求还是远端制品服务

## 常见坑

- 插件失败，但日志只看到表象，没有记录关键上下文
- 日志过度清洗，导致真实错误被吞掉
- 归档路径依赖临时目录，构建后不可复现
