# 生命周期与执行主链路

## Worker 在链路中的位置

- Dispatch 负责把任务分配到合适的执行环境
- Agent 负责构建机侧守护、拉起和环境准备
- Worker 负责实际执行任务并回传结果

## 读取 Worker 的推荐顺序

1. 看启动入口和准备阶段
2. 看领取任务与执行循环
3. 看结束、清理和结果回传

## 主链路心智

- 启动时先拿构建变量并准备工作区
- 启动日志与心跳等基础能力
- 循环领取任务并根据任务类型找到执行器
- 执行结束后回传状态、变量和结果
- 最后做清理与结束处理

## 最值得优先理解的点

- `Runner`：主生命周期控制
- `loopPickup` 一类逻辑：领取并推进任务
- `TaskFactory`：任务类型到执行器的映射
- `ITask`：任务执行器共性抽象

## 常见问题判断

- 任务根本没到 Worker：优先看 Agent / Dispatch
- 任务到了但没有正确执行：优先看 `TaskFactory` 和执行器
- 任务执行了但结果不对：再看变量、日志、归档和 API 回传
