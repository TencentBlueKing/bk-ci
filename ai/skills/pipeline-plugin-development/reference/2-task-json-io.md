# task.json、输入输出与错误码

## `task.json` 的定位

- 它是插件与平台之间的契约
- 它同时影响表单展示、运行时行为和插件执行结果
- 任何会影响使用方体验的字段，都应优先在这里设计清楚

## 输入字段设计重点

- `label`、`type`、`required`、`default` 要表达真实使用方式
- 敏感信息必须标记为敏感字段
- 有条件展示的字段要谨慎使用依赖关系，避免配置爆炸
- 默认值要稳定，不能依赖偶然环境

## 输出字段设计重点

- 输出变量名称要稳定、语义清晰
- 只暴露下游真正会消费的信息
- 不要把临时调试数据做成正式输出
- 输出变化会直接影响其他插件和脚本

## 错误码心智

- 错误码是插件对外的故障分类，不只是内部日志编号
- 要能让用户快速判断是参数问题、环境问题、外部依赖问题还是内部异常
- 同一类问题不要在多个位置随意复用不同错误码

## 校验优先级

1. 先在 `task.json` 做字段约束和基础校验
2. 再在运行时做语义校验和环境校验
3. 对用户可恢复的问题给出明确提示和稳定错误码

## 常见坑

- 默认值和真实运行逻辑不一致
- 输出变量名随实现改动频繁变化
- 敏感信息没标记，导致日志泄漏
- 把错误信息写成纯自然语言，没有结构化错误码
