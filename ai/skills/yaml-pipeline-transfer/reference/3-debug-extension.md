# 表达式、切面、调试与扩展

## 表达式系统

YAML 里常见的不只是静态字段，还包括：

- 变量引用
- 函数调用
- 条件表达式

这意味着字段转换不能只做“文本搬运”，还要确保表达式语义在转换前后保持一致。

高频风险：

- 表达式被提前求值
- 导出后表达式格式变化
- 模板展开后变量作用域变化

## 切面系统

切面通常介入 YAML 转换前后：

- 转换前补充或调整上下文
- 转换后修正模型或 YAML 结果

所以当你看到“相同 YAML 在不同场景下结果不一样”，要优先怀疑：

- 切面包装器
- 上下文差异
- PAC / 模板 / 变量注入差异

## 如何扩展 YAML 字段

新增 YAML 字段时，至少要检查：

1. YAML 数据模型是否支持
2. `yaml2Model` 是否能消费它
3. `model2Yaml` 是否能写回它
4. 模板、表达式、切面是否需要感知它
5. 历史 YAML 与默认值是否兼容

不要只加一边。只支持导入或只支持导出，都会造成双向不一致。

## 调试建议

排查 YAML 转换问题时，建议保留下面几份样本：

- 原始 YAML
- 模板展开后的中间结果（如果有）
- 转换后的 `Model`
- 再导出的 YAML

对比这几份数据，比只盯着一段代码更容易定位问题。

## 常见排查路径

### 问题 1：YAML 能解析，但结果不对

优先检查：

- `ModelTransfer`
- Stage / Container / Element 各自转换器
- 模板和变量是否提前改变了输入

### 问题 2：导出 YAML 丢注释、锚点或格式乱

优先检查：

- `TransferMapper`
- 格式化逻辑
- merge 逻辑

### 问题 3：新增字段后历史 YAML 异常

优先检查：

- 默认值
- 兼容处理
- 双向转换是否同时覆盖

## 性能注意

大型 YAML 文件常见问题：

- 重复解析
- 重复格式化
- 模板嵌套过深
- 表达式和切面链过长

优化时优先确认瓶颈是在：

- 文本解析
- 模板展开
- 模型构建
- 执行链路后续处理
