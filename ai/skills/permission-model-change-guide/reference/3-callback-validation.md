# 回调、验证与常见坑

## 验证顺序建议

1. 先验证资源和动作是否注册成功
2. 再验证实例选择器和申请链路
3. 再验证回调接口和资源查询
4. 最后验证旧数据、国际化和用户组体验

## 回调链路关注点

- IAM 能否正确查询资源实例
- 回调接口是否返回了正确的资源结构
- 用户申请权限时是否能看到正确资源和动作

## 高风险检查项

- 国际化名称是否一致
- 默认用户组和权限模板是否需要同步
- 新资源是否能出现在权限申请、授权、校验和列表查询全链路中

## 常见坑

- 资源定义成功，但实例选择器或回调没接上
- 国际化漏配，导致界面不可读
- 校验只测新建资源，不测历史资源
