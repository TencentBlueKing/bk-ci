# IAM 配置、迁移与初始化

## 一次完整变更通常包含

- 后端枚举或资源类型定义
- IAM RBAC 配置文件
- 初始化数据或迁移脚本
- 线上变更执行方式

## 迁移心智

- 新模型不只服务新数据，还要考虑历史资源如何补齐
- 初始化脚本和线上 API 变更不要互相打架
- 变更顺序要保证中间态也尽量可运行

## 线上落地前要确认

- 是走 SQL / 初始化脚本，还是走 API 接口
- 是否需要多环境同步
- 是否存在重复执行、幂等和版本升级问题

## 常见坑

- 本地只改 JSON，线上没补初始化
- 同一批资源被脚本和 API 双写
- 历史资源没有回填，导致旧数据无权限
